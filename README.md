# VibeColorizer

Автоматически подстраивает цвет анимации «Моей волны» под обложку текущего трека с помощью CSS‑фильтров.

---

## Как работает

1. Аддон загружает обложку трека и извлекает из неё **доминирующий цвет** (одним из трёх методов – см. ниже).
2. Получает **текущий цвет волны** из CSS‑переменной `--vibe-gradient-stop-0` (основной цвет градиента).
3. Вычисляет разницу в цветовом тоне (оттенке), яркости и насыщенности между цветом обложки и цветом волны.
4. Применяет к canvas анимации фильтры `hue-rotate`, `saturate` и `brightness`, чтобы цвет волны стал максимально близок к цвету обложки.

Все изменения происходят **плавно** (длительность настраивается) благодаря CSS‑переходам, что исключает рывки при переключении вкладок.

---

## Настройки аддона

Все параметры доступны в интерфейсе PulseSync (раздел «Настройки аддонов»).

| Параметр | Описание |
|----------|----------|
| **Включить** | Полное включение/отключение аддона. |
| **Максимальная интенсивность** | Сила изменения цвета (0 – выкл, 1 – полное перекрашивание). Интенсивность автоматически снижается, если целевой цвет уже близок к исходному. |
| **Насыщенность волны** | Дополнительный множитель насыщенности (1 – оригинал, >1 – более насыщенная, <1 – более блёклая). |
| **Базовая яркость волны** | Базовая яркость (1 – оригинал, >1 – светлее). К ней добавляется адаптивная коррекция (если включена). |
| **Адаптивная яркость** | Автоматически регулирует яркость волны в зависимости от яркости обложки: светлые обложки → волна светлее, тёмные → темнее. |
| **Метод извлечения цвета** | Выбор алгоритма определения доминантного цвета обложки (см. раздел ниже). |
| **Предпочтение насыщенности** | Влияет на метод извлечения цвета (см. раздел «Методы обработки обложек»). Для стандартного метода – доля самых насыщенных пикселей (0 – все, 1 – только самые насыщенные). Для сбалансированного и альтернативного – баланс между частотой и насыщенностью (0 – только частота, 1 – только насыщенность). |
| **Длительность анимации** | Время перехода при смене цвета (сек). Влияет на плавность смены оттенка. |
| **Спецобработка белых/чёрных/серых обложек** | Для однотонных обложек (белых, чёрных, серых) включает особую логику: белые → волна становится светлее, чёрные → цвет отключается (ч/б эффект), серые → слегка корректируется яркость. Если выключено – цвет волны не меняется для таких обложек. |
| **Предзагрузка обложек** | Загружает обложки следующих треков в очереди, чтобы переходы были плавными. |
| **Не применять фильтр на паузе** | Если включено – на паузе фильтр сбрасывается, волна возвращается к исходному цвету. Если выключено – фильтр остаётся активным. |
| **Режим отладки** | Выводит в консоль подробную информацию о каждом этапе работы (цвета, расчёты, загрузки). Полезно для диагностики. |

## Методы обработки обложек

Аддон предлагает **три способа** извлечения доминирующего цвета из обложки. Выбор метода влияет на то, какой цвет будет считаться «главным» на изображении.

### 1. Стандартный метод (по умолчанию)

- **Принцип:**  
  Все пиксели обложки сортируются по насыщенности (от самой насыщенной к наименее насыщенной). Затем отбирается **topPercent** самых насыщенных пикселей (этот процент зависит от параметра `Предпочтение насыщенности`). Из отобранных пикселей вычисляется средний цвет.

- **Когда использовать:**  
  Хорош для обложек, где есть ярко выраженный насыщенный объект (например, красный силуэт на чёрном фоне). Метод «цепляется» за насыщенные области, даже если они занимают мало места.

- **Особенность:**  
  Может «игнорировать» большие фоновые области, если они имеют низкую насыщенность (серый, чёрный, белый). Даёт яркий, контрастный цвет. В подавляющем большинстве обложек дает отличный результат.

---

### 2. Сбалансированный метод (измененный стандартный)

- **Принцип:**  
  Пиксели группируются по оттенку (с шагом 15°), для каждой группы вычисляется её частота (доля пикселей) и средняя насыщенность. Группа с максимальным взвешенным баллом (`частота × (1 - pref) + насыщенность × pref`, где `pref` – параметр `Предпочтение насыщенности`) выбирается, а её средний цвет становится результатом.

- **Когда использовать:**  
  Универсальный метод, который учитывает и частоту, и насыщенность. Даёт цвет, который одновременно и часто встречается, и достаточно насыщен, чтобы быть заметным.

- **Особенность:**  
  Тестовый метод. Использовать можно на свое усмотрение, т.к. он дает нестандартный но более "правильный" результат.

---

### 3. Альтернативный метод (гистограммный)

- **Принцип:**  
  Цвета квантуются (уменьшается количество уникальных цветов до 16×16×16), строится гистограмма частот. Для каждого кластера рассчитывается комбинированный балл (`частота × (1 - balance) + насыщенность × balance`, где `balance` – параметр `Предпочтение насыщенности`). Выбирается кластер с максимальным баллом.

- **Когда использовать:**  
  Хорош для обложек с большим количеством разных цветов, где нужно выделить наиболее «важный» цвет, учитывая и частоту, и насыщенность.

- **Особенность:**  
  Может «усреднять» цвета в кластерах, что иногда приводит к получению пастельного оттенка.

## Управление исключениями (контекстное меню)

Если для какого-то трека вы хотите отключить автоматическую подстройку цвета (например, обложка слишком тёмная или цвета не сочетаются), вы можете **исключить его**:

- Кликните правой кнопкой мыши по треку в очереди или плеере.
- В открывшемся контекстном меню выберите пункт **«Откл. изм. цвет для трека»**.
- Цвет волны для этого трека сразу вернётся к исходному.
- Для повторного включения выберите **«Вкл. изм. цвет для трека»**.

Список исключённых треков сохраняется в локальном хранилище браузера и не сбрасывается при перезагрузке страницы.

## Технические детали

- **Квантование:** 16 уровней на каждый канал RGB (для альтернативного метода).
- **Размер превью:** 100×100 пикселей (для производительности).
- **Кеширование:** обложки и их доминантные цвета кешируются до 150 записей.
- **Анимация:** CSS-переходы (`transition`) с линейным easing и настраиваемой длительностью (0.1–1.5 сек). Это обеспечивает плавность без использования `requestAnimationFrame`, что устраняет проблемы при переключении вкладок.
- **Наблюдение за цветом волны:** используется `MutationObserver` на корневом элементе для отслеживания изменений CSS‑переменной `--vibe-gradient-stop-0`.
- **Хранение исключений:** список ID исключённых треков хранится в `localStorage` под ключом `VibeColorizer_excluded`.
- **Debounce паузы:** задержка 200 мс при обработке изменений состояния воспроизведения, чтобы избежать ложных срабатываний при перемотке.

## Совместимость

- Работает только с новой страницей «Моя волна» (эксперимент `WebNextNewWaveTab`).
- Не конфликтует с другими аддонами.
- Рекомендуется использовать только с **отключенной** настройкой «Использовать альтернативную цветовую схему Волны» (в настройках Яндекс Музыки).

---

## Примечания

- При первом запуске может потребоваться несколько секунд для загрузки обложек и расчёта цветов.
- В режиме отладки в консоли выводятся подробные логи – используйте для диагностики.
- Если цвет волны временно становится чёрным (например, при переключении вкладок), аддон автоматически повторяет попытку применения фильтра через 30 мс.
